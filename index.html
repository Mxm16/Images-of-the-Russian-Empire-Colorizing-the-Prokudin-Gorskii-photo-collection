Prokudin-Gorskii Image Alignment

Approach

I implemented a two-stage alignment pipeline:

  • Channel Separation: The raw glass plate images were split into three grayscale channels (Blue, Green, Red).
  • Alignment:
    • Small images (.jpg): aligned using an exhaustive search over displacements (±15 pixels) with SSD as the metric.
    • Large images (.tif): aligned using a pyramid strategy. The image was recursively downsampled, aligned at the coarsest level, and refined at higher resolutions.
  • Post-processing: The aligned channels were stacked to form the final RGB image, with pixel values clipped to [0,1] before converting to 8-bit format.

Results on Provided Images

Small Images (.jpg)

  • SSD (Sum of Squared Differences) — This function takes two image patches (reference and candidate), subtracts them pixel by pixel, squares the differences, and sums the result. The smaller the value, the better the alignment. It is simple and efficient when channel brightness is consistent.
  • NCC (Normalized Cross-Correlation) — This function first subtracts the mean from each image patch, then computes the dot product between them, normalized by their magnitudes. The result ranges from -1 to 1, with higher values indicating better alignment. NCC is more robust when exposures have brightness or contrast differences.

Single-Scale Alignment Routine

The align routine is the core exhaustive search method. Its inputs are: a candidate channel (to be shifted), a reference channel, the maximum shift window, and the chosen metric (SSD or NCC).

  1. Crop margins — Both channels are cropped by a fixed number of pixels (default: 50) from all sides. This ensures that border artifacts caused by shifting do not bias the score.
  2. Initialize best score — For SSD, the initial best score is set to infinity (because we want to minimize it). For NCC, it is set to negative infinity (because we want to maximize it).
  3. Exhaustive search — The algorithm tries all integer displacements (dy, dx) within ±15 pixels (by default). For each displacement:
    • Shift the candidate channel using array rolling.
    • Crop the shifted result with the same margin size.
    • Compute similarity score using the chosen metric.
    • If the score is better than the current best, update the best score, record the shift, and keep the shifted image.
  4. Return aligned channel and shift — After the loop finishes, the routine returns the shifted channel and the best (dy, dx).
